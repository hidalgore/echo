# ECHO Web v2 MVP — Acceptance Criteria

## Homepage

- Header includes Home, Events, For Hosts, Nonprofit, How It Works, Trust & Access.
- Primary CTA is Start Hosting.
- Secondary CTA is Explore Events / Find Events.
- Hero phone shows Home / Discover first.
- Hero floating labels are Entry Ready, 21+ Verified, ECHO Circle, Door Mode Ready, and Event Health / Host Dashboard Live.
- Nonprofit is not a main hero label or primary homepage pillar.
- Search command rail appears between hero and events.
- Real public events appear immediately.

## Public Events

- Event cards include title, host, venue/city, date/time, price, verified host badge, age badge when needed, and save/bookmark.
- Donation indicator only appears when the event has an active donation campaign.
- QR codes never appear on public event cards.

## Event Details

- Details / Tickets tab row remains sticky.
- Ticket tiers are visible and selectable.
- ECHO Circle path appears for ticket quantity 2+.
- Refund, transfer, and age rules are visible before checkout.
- Donation section appears only for nonprofit-enabled events.
- Sticky checkout CTA remains available.

## Web Checkout

- Guest checkout is supported.
- Apple Pay, Google Pay, and card payment are supported as design requirements.
- Fees/taxes are clear before payment.
- Promo code support is represented.
- App download is optional after purchase, not required before purchase.
- Full web checkout is possible from day one.

## Age Verification Handoff

- Restricted purchases pause before payment.
- Desktop offers QR, SMS, and email handoff options.
- Checkout state is preserved during handoff.
- Phone verification can complete in app or mobile web.
- Successful verification resumes the same desktop checkout automatically.
- Failed verification blocks checkout.
- No payment is taken before verification.

## ECHO Circle Web

- Organizer can start Circle from web checkout for 2+ tickets.
- Organizer pays for one ticket now.
- Friends claim and pay separately.
- Universal invite link works on web without app install.
- Recipient flow supports age verification before payment when required.
- Recipient confirmation states: Your ticket remains confirmed even if the Circle does not fill.

## Web Wallet Companion

- Active Tickets are visible.
- Saved Events are visible and expired saved events auto-remove.
- Circle status is visible.
- Past Tickets / receipts are visible.
- Add to Apple Wallet / Google Wallet is available where supported.
- QR fallback appears only inside actual ticket fallback view.

## Host Command Center

- Dashboard adapts to Before / Live / After event states.
- Host can create events from web.
- Event Workspace includes Overview, Sales, Guests, Circle, Door, Payouts, Reports, Settings.
- Door tab is a command center/readiness/manual fallback surface; mobile/tablet remains primary scan surface.
- Reports include attendee CSV, attendance CSV, analytics PDF, closeout PDF, donation CSV if applicable.

## Nonprofit Positioning

- Nonprofit appears in nav and For Hosts context.
- Nonprofit is not a homepage headliner.
- Nonprofit tools only appear in creation for verified nonprofit hosts.
- Donation revenue remains separated from ticket revenue.
