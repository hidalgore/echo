# ECHO Ask AI / All Access Search Screen Standard v1

## Purpose
The Search tab is now the Ask ECHO intelligence surface, not a basic keyword search page. It should communicate that ECHO can answer questions about events, tickets, Wallet entry, age verification, ECHO Circle, host support, and escalation paths.

## Locked UX Direction
- Header title: `Ask ECHO`.
- Hero positioning: `ECHO AI` + `ALL ACCESS`.
- Primary promise: `Search events. Solve access issues. Get answers instantly.`
- Main input placeholder: `Ask ECHO anything…`.
- Search copy must encourage natural-language questions, not only keyword search.
- Filters remain available, but secondary to AI intent.

## Required AI Quick Prompts
The screen must expose quick prompts for:
1. Find events
2. My ticket
3. Wallet help
4. Age verification
5. ECHO Circle
6. Host support

## Required Starter Cards
The default state should include AI starter cards for:
- Plan my weekend
- Use my ticket
- Fix an access issue
- Explore by vibe
- Manage ECHO Circle

Each card should include a short, plain-language explanation and route the user into a relevant AI/search query.

## Required Trust + Intelligence Module
The default screen should include an AI answer preview module demonstrating a real access answer, such as offline-ready Wallet entry. This module should include a privacy/trust line and capability pills covering event discovery, Wallet troubleshooting, and age-gated access guidance.

## Results Behavior
After a query, results should be grouped into:
- Events
- AI Help
- Support Escalation

A lightweight ECHO AI Summary card should appear above grouped results to make the experience feel assisted and intelligent.

## Design Standard
- Premium charcoal background.
- ECHO gradient used as restrained edge lighting, not heavy decoration.
- Search input is the primary visual object.
- AI orb/pulse may be used as the signature visual identity.
- QR imagery should not appear on this surface.
- Copy should be calm, trust-first, and operationally clear.

## Future Enhancements
- Logged-in attendee personalization: active ticket, Circle pending, upcoming event help.
- Logged-in host personalization: live event, sales health, Door Mode, closeout report help.
- Voice-first mode with clear privacy language.
- Human support escalation that carries event, ticket, payment, and device context.
